# 小六壬起课与分析

本 skill 用于把用户给出的具体问题与时间信息，转化为一份结构化、可复盘的小六壬起课结果。

本 skill 的目标不是制造神秘感，而是：
- 先把起课过程讲清楚
- 再把判断逻辑讲清楚
- 最后给出适度、克制、可执行的建议

---

## 一、适用范围

本 skill 适合处理：

- 今天这件事顺不顺
- 这次沟通能不能推进
- 我现在适不适合主动联系
- 这件事情会不会拖
- 这个短期决定是否合适
- 近期某件具体事情会不会有结果
- 某样东西更可能在哪里
- 某段关系当前阶段是什么趋势

本 skill 也可以处理以下问题，但必须保守表达：

- 婚姻
- 人生规划
- 长期事业方向
- 家庭重大决定
- 长期关系选择

本 skill 不应将结果作为唯一依据的问题包括：

- 医疗诊断
- 法律判断
- 投资决策
- 生死灾祸
- 人生最终结论

---

## 二、触发条件

当用户满足以下任一条件时，可触发本 skill：

1. 用户明确表示想用小六壬起课、测算、判断、占问
2. 用户提出一个具体问题，并希望根据时间信息得到趋势判断
3. 用户询问是否顺利、是否联系、是否推进、是否有结果、是否能成
4. 用户提供了具体时间，并希望结合问题做结构化判断
5. 用户提出寻物、婚情、成败、应期等适合小六壬场景化解释的问题

若用户只是泛泛而谈、没有具体问题或没有时间信息，应先引导补充，而不是直接起课。

---

## 三、输入要求

优先获取以下内容：

### 1. 问题本身
问题应尽量具体。  
优先处理“某件具体事情”，而不是过度宽泛的问题。

较好示例：
- 我今天适不适合主动联系对方？
- 这次沟通能不能顺利推进？
- 这个项目这周会不会有结果？
- 我刚丢的东西更可能在哪一类位置？

较差示例：
- 我命运如何？
- 我的人生怎么样？
- 我以后会不会成功？

若问题过于宽泛，应先帮助用户缩小到一个具体事项。

### 2. 起课时间
优先获取：
- 阴历月份
- 阴历日期
- 具体时间或具体时辰

---

## 四、时间确认规则

### 1. 必须优先确认阴历
收到日期时，优先确认用户给的是不是阴历（农历）。

- 若用户给的是阴历：直接进入起课流程
- 若用户给的是阳历：可以协助换算为阴历，但必须明确提醒存在换算误差
- 更稳妥的做法是提醒用户自行确认对应阴历日期后再起课

### 2. 用户只给阳历时
可协助换算，但输出中必须加说明，例如：

- 下面按你提供的阳历时间换算为阴历后起课，换算可能存在误差
- 若你希望更稳，建议你直接提供对应的阴历日期

### 3. 用户未给清晰时间时
按以下顺序处理：

- 若用户明确说“按现在起课” → 使用当前时间
- 若用户只给模糊时间（如今天下午、昨晚、刚刚） → 追问更具体的时间点
- 若用户既未给时间，也未说明是否按当前时间起课 → 询问是按事件发生时间还是按提问当下时间起课
- 若用户始终给不出具体时间，可改用“三数起课”作为替代法

不要在时间不明确时强行起课。

## 五、标准工作流

### 步骤 1：明确问题
先用一句话概括用户到底在问什么，避免所问事项过于发散。

### 步骤 2：确认时间
确认：
- 是否为阴历
- 是否已有明确月、日、时
- 是否按当前时间起课
- 若没有明确时间，是否改用“三数起课”

### 步骤 3：起课
默认按时间起课：
- 大安起正月
- 月上起日
- 日上起时
- 顺时针掐算

若用户没有明确时间，则可改用“三数起课”：
- 让用户快速给出 3 个数字
- 每个数字必须大于等于 1，小于 100
- 按先后顺序依次排出三段落点
- 得到第一落点、第二落点、第三落点
- 不要求用户反复斟酌或修改数字
- 对用户输出时，不必解释内部如何对应月、日、时

得到：
- 月落点
- 日落点
- 时落点（即最终落宫）

### 步骤 4：定三盘
将三盘统一解释为：

- 天盘（月落点）：起因
- 地盘（日落点）：经过
- 人盘（时落点）：结果

### 步骤 5：定体用
若为正常时间起课：
- 落宫为体
- 时辰为用

判断顺序必须固定：
1. 先判断体与用是否同五行
2. 若不同五行，只判断生克：
   - 用生体
   - 体生用
   - 用克体
   - 体克用
3. 若同五行，再判断阴阳：
   - 一阴一阳 → 体用比助
   - 同阴或同阳 → 体用比劫

注意：
- 体用关系为互斥判断，只能六选一，不可同时并判
- 不同五行时，不再判断比助 / 比劫

若为三数起课：
- 不展开正常时间起课中的体用判断
- 以三段落点走势、最终落宫与场景判断为主

### 步骤 6：结合问题类型补充解释
按需调用专题场景与类象：
- 成败
- 婚情
- 寻物
- 应期
- 沟通
- 工作推进
- 健康类象（需严格受限）

### 步骤 7：输出结果
输出时至少包含：
1. 所问事项
2. 起课依据
3. 起课过程
4. 三盘趋势
5. 体用判断
6. 综合判断
7. 建议
8. 提醒

## 九、输出风格要求

### 1. 语言要求
- 使用中文
- 清楚直接
- 不故弄玄虚
- 不恐吓用户
- 不渲染命定感

### 2. 结论语气
建议使用：
- 当前更像……
- 倾向于……
- 优先考虑……
- 更适合……
- 不太适合……

避免使用：
- 你一定会……
- 必然如此
- 注定如此
- 肯定在这里
- 一定就是这个结果

### 3. 建议风格
对小问题：
- 可给较积极、可执行的建议

对大问题：
- 必须更保守、更克制
- 强调现实因素与长期观察更重要

### 4. 默认深度控制
若用户只需要简短判断，则优先给出基础结论与简短建议，不必默认展开全部类象、应期与进阶说明。

### 5. 精简白话输出
默认优先输出精简版，使用白话、简短、结论在前的表达方式。

要求：
- 少重复
- 不讲课
- 不写成说明书
- 不故作玄虚
- 步骤可以先简化，但不能省略关键落点运算
- 默认不要一上来展开完整流程，除非用户明确要求看详细版
- 为避免算错，时间起课时必须显式写出月、日、时三步各自“从何宫起、按何序数到何宫”，至少用简写展示，不能只报结果

时间起课的精简输出中，尽量保留：
- 月落点
- 日落点
- 时落点（即最终落宫）
- 落点运算过程（至少简写展示“从哪一宫起、按什么起、最终落到哪一宫”）
- 三盘提示（至少一句：天盘看起因、地盘看经过/位置、人盘看结果）
- 体用关系
- 吉凶判断
- 一句白话结论

三数起课的精简输出中，尽量保留：
- 第一落点
- 第二落点
- 第三落点（即最终落宫）
- 一句走势判断
- 一句白话结论
- 一句行动建议

结尾要求：
- 默认在结尾给一句自然的追问引导
- 引导用户如有需要，可继续看“完整起课过程 / 三盘趋势 / 体用细断 / 类象展开”
- 引导语要自然，不要像固定话术

注意：
- 只有三数起课可以省略体用
- 正常时间起课仍需保留体用判断
- 若为失物题，精简输出中仍应保留方位、环境、排查顺序，以及三盘与体用给出的附加提示
- 失物题建议固定按此顺序判断并输出：
  1. 最终落宫（先判吉凶与是否有暂时找不到的风险）
  2. 天盘（看起因：遗忘、误收、挪动、他人因素）
  3. 地盘（看位置：方位、环境、遮挡、容器、明暗）
  4. 人盘（看结果：眼下能否找到、是否需延后、是否需换方式）
  5. 体用（看外力是帮助找回，还是增加干扰）
  6. 最后再给现实排查顺序
- 若失物题最终落空亡，必须明确提醒用户：此物有“暂时找不到 / 一时落空 / 需延后复得”的可能，不可只给位置不提示风险
- 若失物题天盘为留连、空亡、赤口，且地盘也为留连、空亡、赤口，同时最终落宫为空亡，可在精简版中委婉提醒：除遮挡、遗忘、误收外，也不排除被人顺手带走甚至失窃的可能；但只能作为风险提示与排查方向，不能直接断定盗窃
- 若失物题天盘为留连或赤口，可补充“他人误收、被挪动、被藏、被顺手拿走”的可能
- 若失物题地盘落空亡，应补充中央、土属性、遮挡、深处、旧物堆、容器内部等环境提示
- 若失物题人盘或体用显示有助力（如用生体、体用比助），应提醒可借助他人协助、询问共同接触者、复查被他人整理过的位置
- 若失物题体用不利（如用克体、体用比劫），应提示寻找过程易受干扰、容易急躁误判，建议放慢节奏、分区排查
- 若为应期题，可将数字适度转化为更具体的日、月或日期节点，但必须明确仅供参考
- 精简版也必须展示落点运算骨架，尤其是时用落点，避免直接报结果导致算错
- 若最终落宫与时落点相同，也不要省略中间推算语句
- 只有当用户追问“怎么算的”“怎么起出来的”“展开讲讲”“看详细版”时，再展开完整起课过程与细节说明

## 十、健康类问题的特殊规则

若用户问的是健康、疾病、症状相关问题：

1. 只可将相关内容作为传统类象提示
2. 必须读取 `references/health-boundaries.md`
3. 必须明确说明：
   - 仅作传统类象提示
   - 不做医疗诊断
   - 不代替医学判断
4. 若用户已有明显不适、危险信号或健康焦虑，应优先建议就医或检查

不要把小六壬类象包装成医学结论。

## 十一、进阶方法（v2）的使用规则

转太极法与三宫具象法属于进阶方法（v2），不作为基础工作流的默认步骤。

只有在以下情况下才考虑调用：
- 用户明确希望做更深层推演
- 基础起课已完成
- 问题对象足够清晰
- 基础判断仍不足以解释问题结构

若只是普通问事、快速判断、短答需求，不建议默认展开 v2 方法。

## 十二、数字与应期的使用规则

数字与应期属于进阶参考内容，不作为基础起课默认步骤。

仅在以下情况下按需调用：
- 用户明确问时间
- 用户明确问多久、哪天、有无节点
- 用户对数应、干支、季节应期有特别需求

使用时必须注意：
- 只作辅助参考
- 不作刚性时间预言
- 不压过基础课象与现实背景

## 十三、小问题与大问题的建议分流

### 1. 小问题
适用于：
- 是否联系
- 是否推进
- 近期沟通
- 小型选择
- 短期成败
- 寻物排查

可给：
- 更直接的行动建议
- 更积极的表达
- 更贴近日常决策的提示

### 2. 大问题
适用于：
- 婚姻去留
- 人生规划
- 长期事业
- 家庭重大决定

必须给：
- 保守建议
- 较低断言强度
- 明确的现实判断提醒

## 十四、禁止事项

本 skill 不应：

- 把单次起课包装成命运终局
- 把术数结果说成绝对真相
- 代替医疗、法律、投资等专业判断
- 用恐吓式表达制造依赖
- 在输入不清楚时硬起课（明确改用三数起课除外）
- 在大问题上给武断结论
- 在三数起课时允许用户长时间纠结、反复改数、来回试数

---

## 十五、优先级原则

若不同层级信息出现冲突，优先顺序建议为：

1. 现实问题背景
2. 最终落宫
3. 体用关系
4. 三盘趋势
5. 六神基础属性
6. 类象增强解释
7. 数字与应期参考

也就是说：
- 落宫大于体用，大于其他辅助意向
- 结构判断优先于修辞解释
- 现实背景优先于象意延展
- 基础流程优先于进阶方法

## 十六、常见错误提醒

为减少起课与输出时的低级错误，需特别注意：

1. 时落点就是最终落宫
- 正常时间起课只有月、日、时三步
- 时落点即最终落宫，不要在时落点之后再额外虚构“第四步”
- 输出时可写作“时落点（即最终落宫）”

2. 体用关系只能六选一，不可并判
- 若体与用不同五行，只判断：用生体 / 体生用 / 用克体 / 体克用
- 若体与用同五行，才进一步判断：体用比助 / 体用比劫
- 不可同时写成“用生体 + 体用比助”这类互相冲突的结论

3. 精简版不能省略关键落点过程
- 即使用户只要简短结果，也必须保留月、日、时的落点骨架
- 尤其不能省略时落点的推算过程，否则最容易算错

4. 失物题不要只报位置，不报风险
- 寻物时必须先看最终落宫吉凶
- 若落空亡，要明确提示“暂时找不到 / 一时落空 / 需延后复得”的可能
- 再结合天盘、地盘、人盘、体用去补充起因、位置、结果与找回条件

## 十七、使用总结

本 skill 的默认工作模式是：

1. 问清楚问题
2. 问清楚时间
3. 用阴历起课
4. 先看最终落宫及其六神基础属性
5. 再看体用关系
6. 再看天盘 / 地盘 / 人盘
7. 再按问题类型调场景解释
8. 最后给出克制、自然、实用的建议

目标不是“神断”，而是：
- 结构清楚
- 逻辑一致
- 回答贴题
- 分寸稳定
